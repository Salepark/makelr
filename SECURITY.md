# Security Policy

## Supported Versions

We take security seriously. The following versions of Makelr are currently supported with security updates:

- `main` branch (latest)
- The most recent tagged release

Older versions may not receive security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability in Makelr, **please do not open a public issue**.

Instead, report it responsibly using one of the following methods:

- Email: security@makelr.com (or replace with your actual contact email)
- Or: Send a private message to the maintainers via GitHub

Please include as much detail as possible:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fix (if you have one)

We will acknowledge receipt of your report as soon as possible and work on a fix.

---

## Disclosure Policy

- We will investigate all legitimate reports.
- We will aim to provide a fix or mitigation as quickly as reasonably possible.
- We may coordinate with you on responsible disclosure timing.
- We will credit reporters who wish to be acknowledged (unless you prefer to stay anonymous).

---

## Scope

This security policy applies to:

- The Makelr core application
- Official plugins and integrations in this repository
- Build and deployment scripts included in this repository

Third-party services, user-provided LLM APIs, and external sources are **out of scope**, but we appreciate reports if you believe an integration creates a serious risk.

---

## No Bug Bounty Program

At this time, Makelr does not operate a paid bug bounty program.  
However, we deeply appreciate responsible disclosure and community help in keeping the project safe.
